how does js work

Execution context

As a frontEnd developper you often think about how does js work and how does it do what it does?
First of all js is not understood by browsers and so it does all that it does using a js engine. The right question now to have is how does a JS engine understand the code that is given to it and how does it work on it ?

It first creates a global execution context. But what is an Execution context ? an execution context simply put is a chart that has two columns the Memory and Execution we fill those columns following two phases.
1. First phase is Initial phase in which we initialise all global variables to "undefined" by putting them into the Memory column, when we enconter a function we put their declaration into the Memory column as well
2. Second phase is Execution phase in which we put the line of code related to the variables we've just initialised

Now that we have the lines of code in the column of Execution whenever we enconter a function call we create a function execution context which follows the same steps except it belongs to a function and not the window object

Execution callstack

Now that we now the gist of how an execution context is created, we move into how it is managed. As mentionned above the global execution context is the first to be created, but there is more to this because after creation it stores it in an execution stack which is a type of data structure. As we said whenever a function call is encountered in the lines of code it creates an execution context where does it go ? Onto the stack, on top of the global execution context and then whenever the function is executed the engine pops it out of the stack
